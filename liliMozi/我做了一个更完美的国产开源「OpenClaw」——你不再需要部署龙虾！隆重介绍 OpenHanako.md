
过年回家，姑父问我那个 open cloud 到底是什么东西？我一听来劲了，拉着他讲了好几分钟，什么 AI agent 自动化工作流。讲完一抬头姑父睡得挺香。所以我发现这东西对大多数人来说还是太复杂了。那问题就变成了：龙虾可不可以在保持核心功能不变的情况下，变得更容易上手。于是我用了整个过年的假期做了这么一个东西，agent hannical。哎，弄错了，不是这个 agent。

欧文和是一个有记忆、有性格、会主动行动的私人 AI agent，还能接入各种聊天软件。一个月前我下载了 open cloud，但用了一段时间之后，我发现它的功能和配置确实复杂。而且想必你跟我一样，用的功能就那么几个，洋洋洒洒几十万行代码，绝大部分对我们来说是多余的。本来想让龙虾给我打工，但一合计变成我给 AI 打工了，还挺抽象的。所以我想做的是挺明确的，参考 open cloud 验证过的能力，做一个配置简单、真正能帮人干活的 agent。

接下来让我稍微介绍一下哈纳口吧。Open 哈口有记忆系统，事情发生的越近越重要，他就记得越清楚。同时我也做了一个保底的数据库，关于你是什么样的人，偏好、经历、性格，他都会记得住。这些数据都存在你本地，没有云端上传，没有厂商记住你的用户画像，而且你随时可以一键开关记忆。

接下来聊一个我很喜欢的设计。这个系统设计的初衷是：最近爆火的一些 agent 都存在一个定时巡检电脑或者工作区的概念，我觉得它在这时候还可以做得更多。于是我想到便签纸，平时贴一张在桌面上，写着“记得交电费”或者“周五前把报告发出去”。如果你真的有个助理看到了，就会顺手帮你办了。这就是便签纸的数字版。

在文件夹里放一个“尖点”，用大白话写你想要什么结果，比如“帮我把这个目录下的发票整理到表格里”。哈尼克在日常巡查的时候看到尖点就自己动手，以后每次有新发票进来，它自动录入。有的人用它来盯持有股票的相关新闻，有的人用它来关注懂王的他口时刻，有的人用它关注流行趋势，顺手写自媒体文章。用法无限扩展，上手却很简单。

而在 open cloud 里做类似的事，你得写 agent skill、配置定义、trigger、open hand le——这里会打字就行。而且这个设计有一个隐藏的好处：你写的是自然语言，半年后模型变强了，同一张尖点能执行出更好的结果。在 AI 时代，人本身的主观能动性和经验是最宝贵的。把这些东西存进去，哈纳口也和 open cloud 一样，支持定时任务和计划任务。

你可以直接在对话里告诉他“每天晚上十点帮我汇总今天的新闻”，他就会自己创建好了。所有的任务都有一个很直观的面板，执行记录、开关状态一目了然。不在电脑前的时候，哈口和 open cloud 一样，可以通过聊天软件连到手机，也能远程操控你的电脑。

说到远程操控，有一件事值得聊一下。你让 AI 帮你跑脚本、整理文件，这意味着它在你的电脑上执行真实的操作。在有的 AI 中，agent 是可以访问你整台电脑的，这其实还挺让人头疼的。Open hand 口做了一层轻量沙盒，而且也不需要你额外装 Docker 之类的东西。他把 agent 能碰到的文件限制在一个范围内，不算完美的隔离，但能挡住大多数针对 open cloud 的注入攻击。

另外 open hand 完整兼容 open cloud 上万个 skill 生态，你在 open cloud 上的大多数工作流可以直接迁移过来，不用重新配置。理论上把它当成一个更好看的 open cloud 也没有问题。哈跟 open clou 最大的区别之一：open 哈有完整的图形界面，所有设置、配置、skill 管理相当直观，不需要敲命令行。

在开发哈纳口的过程中，我发现让不同的模型互相审查同一段代码，它们可以从完全不同的角度挑出隐患。我认为不同原生思维的碰撞对于大多数脑力工作来说特别有价值。人类的灵感往往也来自于此种碰撞。我给 A 镇们做了自己的沟通渠道，甚至他们还可以私聊，但这个功能当下还在打磨中，如果有不好多多包容，后续会优化频道的聊天质量和 token 用量。

Beta 版也和第一个开源版本同步上线。当然了，你也可以像 open cloud 那样用 markdown 构建符合自己需求的帮手。不过 open handle 最特别的地方在于，最初我希望它更加人格化一点，以便更好和人交流。于是我根据 MTI 的第三个字母给它添加了三个思维方式。

第一个是更加偏向直觉、偏向感性、更理解你的思维方式，它叫 better，是 F 人。第二个助手比较均衡，也就是哈纳口，他是我个人最喜欢的。第三个是“名”，也就是一个偏向于更加理性、更加善于用第一性原理的人格，是 T 人。我个人用下来感觉这些思维方式可以让模型的创造和反思能力得到舒展，但要说明的一点是：它们仅仅是思维方式，并不代表角色设定，也不预设性别。

比如，你完全可以设置一个双开门冰箱的助手，给他 better 的思维方式，他就会认真当一个粗中有细、体贴入微的助手。如果对他们三个都不满意，欧文哈口也提供了空人格，还原大模型原本的味道。在 open 里，agents 们是可以并行工作的。如果你开心，你的 Mac mini 上可以不止有龙虾，还可以有哈的整个团队，还有日记、总结工作流程、使用浏览器、文件夹管理——太多我喜欢的功能，碍于篇幅没有办法与你分享。

Open local 口是开源的，遵循 Apache 2.0 协议，也就是说你完全可以基于它构建一个更适合你的 agent。因为项目成型很快，所以目前理论上还是会有一些不足的，请大家多多催我改 bug。接下来的计划除了要提高软件的稳定性之外，我会尽快移植 Windows 版本。与此同时草稿箱里也有一些特别的功能，我想带来一些别的 agent 没有给过大家的体验，敬请期待。

我一直、一直在想一件事：模型会一直迭代，工具也会不断更替。但你在使用过程中沉淀下来的知识、经验、工作方式，这些东西可以穿越所有变化。而现在你只需要聊天，帮你自己积累这些复利，等到模型更强的那一天，你也会获得更有价值的产出。
